# Week 4: Python, Automation, and Computational Thinking

## Week Goal

Build enough logic and coding confidence for learners to understand workflows, automation, rule-based systems, and local AI experimentation.

## Concepts Explorer Topics

### Computational Thinking
Break large tasks into smaller, manageable steps.

### Variables and Conditions
Variables store information. Conditions help a program choose what happens next.

### Loops and Functions
Loops repeat steps. Functions package reusable logic.

### Lists and Simple App Logic
Lists help programs manage groups of items such as questions, tasks, or replies.

### Automation and Rule-Based Chatbots
Automation reduces repetitive work. Rule-based chatbots use fixed logic instead of learned language understanding.

### Open-Source Models, Ollama, and Local Inference
Some AI models can run locally on a device using tools such as Ollama, which changes privacy, cost, and hardware tradeoffs.

## Core Ideas

- coding is structured instruction, not mystery
- automation starts with logic
- rule-based systems and AI systems are not identical
- local models can be useful when privacy or cost matters

## Class Activities

### Activity 1: Human Robot Instructions
Students give exact instructions to a human “robot” and discover where logic becomes ambiguous.

### Activity 2: Workflow to Pseudocode
Small groups convert a student workflow, such as assignment submission or revision planning, into plain-language pseudocode.

### Activity 3: Rule-Based Chatbot Sketch
Teams design a subject-help chatbot with conditions, responses, and failure cases.

### Activity 4: Automation Mapping
Students choose a repetitive student task and map which steps could be automated with simple logic.

### Activity 5: Run a Local Model Discussion Lab
The class compares local model use with hosted AI tools and discusses privacy, speed, device limits, and when Ollama-style workflows make sense.
