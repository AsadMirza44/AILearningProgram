# Week 2: Prompt Engineering and AI for Learning

## Week Goal

Teach learners how to communicate with AI clearly, improve prompts step by step, and understand why token limits and generation settings affect results.

## Concepts Explorer Topics

### Prompt
A prompt is the instruction or input given to an AI system.

### Prompt Structure
Strong prompts usually include role, task, context, constraints, output format, and sometimes examples.

### Iteration
Good prompting is a revision process. Learners improve output by improving instructions.

### Verification
Even a well-structured prompt can still produce mistakes, so important claims must be checked.

### Tokens, Context Windows, and Inference
Models read and produce tokens. They also have limits on how much context they can consider at one time.

### AI Next-Token Generation
Language models generate output one token at a time by predicting what should come next.

### Attention, Temperature, Top-p, and Top-k
These settings shape how focused, stable, creative, or variable the model output becomes.

## Core Ideas

- Strong prompting is structured communication.
- Long prompts can exceed context limits.
- Generation settings change the style and stability of output.
- Good prompts still require verification.

## Class Activities

### Activity 1: Weak Prompt Repair
Give teams poor prompts such as `Help me study history`. Teams rewrite them using role, task, context, constraints, and output format.

### Activity 2: Prompt Ladder Challenge
Students improve the same prompt in three rounds. Each round adds one layer: audience, format, and constraints. They compare the outputs after each round.

### Activity 3: Context Window Compression
Groups receive an overloaded prompt with too much information. They must compress it into a cleaner version that preserves what matters most.

### Activity 4: Generation Settings Comparison
Show one factual task and one creative task. Students decide when low temperature or higher variation is more appropriate and explain why.

### Activity 5: AI Tutor Roleplay
One student plays the learner, one plays the AI, and one plays the verifier. The group judges whether the final answer is useful and trustworthy.
