# AI-Enabled Student Program

## Program Summary

**Format:** 6-week interactive AI enablement program  
**Audience:** students and beginner AI learners  
**Delivery Style:** concept-first, activity-led, beginner-friendly, tutor-guided  
**Current App Structure:** dashboard, week pages, concept explorer, class activity studio, image placeholders, checkpoint quiz, reflection

This project now delivers a focused 6-week AI learning journey inside one interactive application. The experience is built around expandable concepts, collaborative class activities, image/GIF placeholders for future media, reusable quiz interactions, and activity launch hooks for future in-app simulations.

## Program Outcomes

By the end of the program, learners should be able to:

- explain core AI, ML, RAG, MCP, prompt, and model concepts in simple language
- use AI tools critically rather than passively
- improve prompts using context, constraints, structure, and iteration
- identify hallucinations, weak outputs, bias, and review points
- understand basic machine learning, regression, neural networks, and local-vs-cloud AI tradeoffs
- understand how retrieval, embeddings, vector databases, and tool-connected systems work
- reason about chatbots, agents, frontier models, open-source models, Ollama, parameters, MoE, and AI cost
- design and present a practical AI-enabled capstone

## App Delivery Model

Each active week in the app uses the same learning workflow:

1. Week hero and objectives
2. Concepts Explorer
3. Core Ideas
4. Images
5. Class Activity Studio
6. Launch Activity placeholder
7. Checkpoint quiz with click-to-reveal answers
8. Reflection

## Current 6-Week Structure

## Week 1: AI Foundations and AI Literacy

Focus:
- what AI is
- what AI is not
- AI in daily life
- multimodal and document AI basics
- hallucinations, bias, and responsible AI
- neural-network intuition as a bridge to modern AI systems

Interactive direction:
- identify AI around us
- compare strong vs weak AI output
- debate trust, safety, and human judgment

## Week 2: Prompt Engineering and AI for Learning

Focus:
- prompt structure
- role, task, context, constraints, and output format
- iteration and verification
- tokens, context windows, inference
- next-token generation
- attention-related generation settings such as temperature, top-p, and top-k

Interactive direction:
- improve weak prompts
- compare output quality from different prompt styles
- observe how generation settings change results

## Week 3: Data Thinking and Machine Learning Basics

Focus:
- datasets, features, labels
- training and testing data
- classification and recommendation
- regression
- bias and overfitting
- neural networks at a conceptual level

Interactive direction:
- identify features and labels
- sort tasks into classification vs regression
- reason about fairness and data quality

## Week 4: Python, Automation, and Computational Thinking

Focus:
- decomposition
- variables, conditions, loops, functions, and lists
- rule-based chatbot logic
- workflow automation
- open-source models
- local inference
- Ollama

Interactive direction:
- break tasks into logic steps
- build lightweight chatbot logic
- discuss when local models are useful

## Week 5: RAG, MCP, and AI-Powered Systems

Focus:
- retrieval
- semantic search
- embeddings
- vector databases
- RAG
- chunking, reranking, citations, and source freshness
- RAG vs fine-tuning
- MCP
- agents vs chatbots
- multimodal and document AI workflows
- frontier models, open-source models, parameters, MoE, and AI cost

Interactive direction:
- design a filesystem-based RAG workflow
- compare grounded vs ungrounded answers
- design a tool-enabled school assistant

## Week 6: Capstone Development and AI Showcase

Focus:
- problem statement
- workflow design
- prompts, tools, and knowledge use
- model choice
- frontier vs open-source tradeoffs
- inference cost and deployment decisions
- safety, testing, and presentation

Interactive direction:
- map the capstone workflow
- justify model and system choices
- present a practical AI-enabled solution

## UI and Content Principles

- keep concept explanations concise and beginner-friendly
- avoid duplicate note-heavy sections
- use image/GIF placeholders where future visuals will appear
- separate concept-level media from week-level visual galleries
- connect every class activity directly to the concept explorer
- keep launch hooks ready for future interactive modules
- require quiz completion before submission, then reveal answers only when requested
