# Week 6: Capstone Development and AI Showcase

## Week Goal

Help learners combine prompting, data thinking, retrieval, tools, model choice, and responsible AI into one practical capstone.

## Concepts Explorer Topics

### Problem Statement and User Persona
Every capstone starts with a real user need and a clearly defined audience.

### AI Workflow
The workflow should show inputs, prompts, data or knowledge, tools, outputs, and review points.

### RAG and MCP in a Capstone
Learners should explain how their solution uses trusted knowledge and structured tool access when needed.

### Model Choice, Frontier vs Open-Source, and Parameters
Learners should justify why a hosted frontier model, a smaller open-source model, or a hybrid setup fits the task best.

### Cost, Inference, and Deployment Tradeoffs
Good projects consider token usage, hardware limits, privacy, and recurring cost.

### Agents, MoE, and System Scale
Learners should understand that larger AI systems combine model architecture and workflow design choices.

### Safety, Ethics, and Testing
Responsible projects include privacy, review, bias checks, failure handling, and clear limits.

## Core Ideas

- start from the problem, not the tool
- explain why each AI component exists
- model choice is a tradeoff, not a popularity contest
- safe systems need testing and human review

## Class Activities

### Activity 1: Capstone Mapping Wall
Teams map their capstone on a wall using cards for user, problem, prompts, knowledge, tools, outputs, and review steps.

### Activity 2: Model Choice Pitch
Each group explains why its solution should use a frontier model, an open-source model, or a mixed approach.

### Activity 3: Risk Review Circuit
Groups rotate and review each other’s projects for privacy, hallucination risk, weak retrieval, unclear tool use, and cost issues.

### Activity 4: Demo Flow Rehearsal
Students practice explaining their project in under two minutes with a clear beginning, system flow, and safety note.

### Activity 5: Future Version Brainstorm
Each team lists one improvement for visuals, one for interactivity, and one for system reliability.
